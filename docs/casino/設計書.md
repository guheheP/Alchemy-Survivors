## 1. 設計の鉄則

1. **完全疎結合**: `src/game/casino/` 配下に全機能を閉じ込める
2. **単方向依存**: `casino → 本体` のみ許可。本体 → casino の参照は `index.js` 経由のみ
3. **ゴールド以外に触れない**: `InventorySystem.addGold` / `spendGold` 以外の本体APIは使わない
4. **削除可能性**: フラグoff → ディレクトリ削除 → 本体が即座に元通りに動く
5. **CASINO_ENABLED と CASINO_VISIBLE の分離**: 機能有効性とUI露出を独立管理

export const CASINO_ENABLED = true;
export const CASINO_VISIBLE = false; // 初期リリース時まで false
export const CASINO_VERSION = 1;

export const EXCHANGE_RATE = {
  goldToMedal: 1,
  medalToGold: 1,
  fee: 0,
  minExchange: 10,
};

export const BET_PER_GAME = 3; // 3枚固定

/**
 * 実効可視性判定（開発者オーバーライド込み）
 */
export function isCasinoVisible() {
  if (!CASINO_ENABLED) return false;
  if (CASINO_VISIBLE) return true;
  try {
    if (typeof localStorage !== 'undefined' && localStorage.getItem('casino_visible') === '1') return true;
    if (typeof window !== 'undefined') {
      const params = new URLSearchParams(window.location.search);
      if (params.get('casino') === '1') return true;
    }
  } catch { /* ignore */ }
  return false;
}

## 4. CasinoManager（エントリ集約）

```javascript
/**
 * src/game/casino/CasinoManager.js
 * シングルトン。カジノ機能全体のライフサイクルを管理。
 */
import { CasinoState } from './state/CasinoState.js';
import { CasinoScreen } from './CasinoScreen.js';
import { pickDailySetting } from './util/dailySetting.js';

export class CasinoManager {
  static _instance = null;
  static getInstance() {
    if (!this._instance) this._instance = new CasinoManager();
    return this._instance;
  }

  constructor() {
    this.inventory = null;
    this.state = new CasinoState();
    this.screen = null;
  }

  init(inventory) {
    this.inventory = inventory;
  }

  serialize() {
    return this.state.toJSON();
  }

  hydrate(data) {
    if (!data) return;
    this.state.fromJSON(data);
    // 日付が変わっていたら設定を再抽選
    const today = new Date().toISOString().slice(0, 10);
    if (this.state.lastSettingDate !== today) {
      this.state.currentSetting = pickDailySetting(today);
      this.state.lastSettingDate = today;
    }
  }

  mountLobby(container, inventory) {
    if (!this.inventory) this.inventory = inventory;
    if (this.screen) this.screen.destroy();
    this.screen = new CasinoScreen(container, this);
    this.screen.render();
  }

  // 両替APIは CasinoScreen 経由で呼ばれる
  exchangeGoldToMedals(amount) {
    if (!this.inventory.spendGold(amount)) return false;
    this.state.medals += amount;
    return true;
  }

  exchangeMedalsToGold(amount) {
    if (this.state.medals < amount) return false;
    this.state.medals -= amount;
    this.inventory.addGold(amount);
    return true;
  }
}
```

## 5. 状態機械（StateMachine.js）

### 5.1 状態一覧

| Phase | 説明 |
|-------|------|
| `NORMAL` | 通常時 |
| `ZENCHO` | 前兆（5〜15G） |
| `CZ` | チャンスゾーン（10G間） |
| `BONUS_STANDBY` | BONUS内部成立後、図柄揃い待ち。他抽選は停止 |
| `BONUS` | BONUS消化中（疑似遊技、青7チャレンジ抽選） |
| `ART` | ART消化中 |
| `TENJOU` | 1200Gハマリ救済モード |

### 5.3 遷移条件

#### BONUS抽選（常時稼働）
- **`BONUS_STANDBY` / `BONUS` 以外のすべての状態** で毎ゲーム BONUS（big/reg）抽選
  - NORMAL / ZENCHO / CZ / ART / TENJOU で同確率
  - 当選時、**現在の状態を `resumePhase` に保存** し `BONUS_STANDBY` へ遷移
- BIG: 約1/300、REG: 約1/450（設定差あり）

#### BONUS_STANDBY（内部成立後の待機状態）
- 目的: BONUS図柄を揃えるまで待機。プレイヤーに告知しつつ緊張感を演出
- **他の抽選は全停止**:
  - ❌ CZ抽選なし
  - ❌ 前兆抽選なし
  - ❌ 天井カウント停止（BONUS成立済みなので不要）
  - ❌ ART抽選もなし（ART中で内部成立した場合、ART残G数の消化も凍結）
  - ✅ 小役抽選は継続（ベル、リプレイ等は出る）
- **告知演出**: レア役（チェリー／スイカ／チャンス目）とリーチ目の出現率を大幅アップ
  - 通常時のレア役確率の3〜5倍程度に強化
  - 「リーチ目」: 通常時には出ない特定の絵柄並び（例: 中段に赤7テンパイで右リール下段に赤7停止など）
  - これにより「ボーナス確定」の視認性を確保
- 遷移: プレイヤーがボーナス図柄（BIG=赤7揃い／REG=赤赤青）を揃えた瞬間に `BONUS` 状態へ

#### BONUS（疑似遊技）
- 規定ゲーム数（BIG=30G、REG=10G）の疑似遊技
- 毎ゲーム **青7フラグ抽選** が走る
- 青7フラグ成立ゲームでは、**目押し不要で自動的に青7が揃う演出** が発動
  - 専用のリール停止演出（全リールが青7で止まる）
  - プレイヤーは見守るだけ
- BONUS中の1ゲームでも青7が揃えば、その BONUS全体で `blue7Succeeded = true` フラグが立つ
- 消化完了時に BONUS終了ロジックへ

#### ART抽選（2経路のみ）
ARTは以下の経路でのみ付与される。BONUS単体ではART突入しない。

1. **CZ → ART**: CZ成功率 約40%（設定により30〜55%）
2. **BONUS中の青7成立 → ART**: BONUS中のレバーON時に青7フラグ抽選で成立

**トータルART期待度（1ボーナスあたり）**: 全設定で **25〜50%** 範囲

#### その他遷移
- `NORMAL → ZENCHO`: チャンス目・弱チェリー等のレア役で約1/15抽選
- `ZENCHO → CZ`: 前兆成功率 約60%（設定差あり）
- `ZENCHO → NORMAL`: 失敗 約40%
- `CZ → ART`: CZ成功時
- `CZ → NORMAL`: CZ失敗時
- `ART → NORMAL`: ART消化終了（残G数が0）
- `NORMAL → TENJOU`: 通常G数 ≥ 1200
- `TENJOU → BONUS_STANDBY`: 次ゲームで強制BIG/REG内部成立

### 5.4 BONUS終了時の復帰ロジック（仕様確定）

```javascript
onBonusEnd(state):
  if (state.blue7Succeeded) {
    // 青7成立 → ART付与
    if (state.resumePhase === 'ART') {
      // ART中BONUS → ART継続＋100G上乗せ＋ストック
      state.art.remainingGames += 100;          // 固定100G
      state.art.stocks += 1;
      state.phase = 'ART';
    } else {
      // 新規ART突入
      state.art.remainingGames = INITIAL_ART_GAMES;  // 例: 50G
      state.phase = 'ART';
    }
  } else {
    // 青7未成立 → 上乗せなし
    if (state.resumePhase === 'ART') {
      // ART残G数はBONUS中凍結されていたため、そのまま復帰
      state.phase = 'ART';
    } else {
      state.phase = 'NORMAL';
    }
  }
  state.blue7Succeeded = false;
  state.resumePhase = null;
```

### 5.5 ART中BONUS 上乗せ仕様（簡潔版）

| 条件 | ART残G数の扱い | 追加上乗せ | ストック |
|------|----------------|-----------|----------|
| 青7成立（BIG/REG問わず） | 凍結→復帰 | **+100G固定** | +1 |
| 青7不成立 | 凍結→復帰 | なし | なし |

- BIG/REGの種別は上乗せ量に影響しない
- 上乗せはBONUS終了後のART復帰時に一括反映

### 5.6 実装方針

純粋関数 `step(state, rngValue, input) → { nextState, events }` として実装。BONUS抽選は状態遷移関数の **最初** に独立して実行。

```javascript
/**
 * @param {SlotSessionState} state
 * @param {number} rngValue
 * @param {GameInput} input
 * @returns {{ nextState: SlotSessionState, events: SlotEvent[] }}
 */
export function step(state, rngValue, input) {
  // 1. BONUS抽選（BONUS_STANDBY / BONUS 中は走らせない）
  if (state.phase !== 'BONUS_STANDBY' && state.phase !== 'BONUS') {
    const bonusFlag = rollBonusFlag(state.setting, rngValue);
    if (bonusFlag !== 'none') {
      return enterBonusStandby(state, bonusFlag);
    }
  }

  // 2. 現在のphase固有の抽選・遷移
  switch (state.phase) {
    case 'NORMAL':        return stepNormal(state, rngValue, input);
    case 'ZENCHO':        return stepZencho(state, rngValue, input);
    case 'CZ':            return stepCz(state, rngValue, input);
    case 'BONUS_STANDBY': return stepBonusStandby(state, rngValue, input); // 告知演出＋図柄揃い判定
    case 'BONUS':         return stepBonus(state, rngValue, input);        // 青7チャレンジ含む
    case 'ART':           return stepArt(state, rngValue, input);
    case 'TENJOU':        return stepTenjou(state, rngValue, input);
  }
}
```

## 6. リール抽選アルゴリズム

### 6.1 1ゲームのフロー

```
1. BET
   ├ medalsSpend(3)
   └ state.gameCount++

2. 内部抽選 (SlotEngine.draw) — フェーズ別に段階実行
   ├ (A) BONUS抽選 (BONUS_STANDBY / BONUS 中は スキップ)
   │    ├ rnd1 = rng.nextInt(0, 65535)
   │    └ bonusFlag = BONUS_PROB_TABLE[setting] → 'big' | 'reg' | 'none'
   │    → 成立時: state.phase = 'BONUS_STANDBY', state.standbyBonusKind = bonusFlag
   ├ (B) BONUS中のみ: 青7チャレンジ抽選
   │    ├ rnd2 = rng.nextInt(0, 65535)
   │    └ blue7Flag = BLUE7_PROB_TABLE[setting][bonusKind] → 'blue7' | 'none'
   └ (C) 小役抽選 (全状態)
        ├ rnd3 = rng.nextInt(0, 65535)
        └ smallFlag = SMALLROLE_PROB_TABLE[setting][phase] → 'bell' | ...
        ※ BONUS_STANDBY では STANDBY専用テーブルを使用（レア役・リーチ目の頻度UP）

3. レバーON → リール全回転開始

4. 各STOP押下 (ReelController.stop)
   ├ 現在の目押し位置 targetIndex を取得
   ├ 滑りコマ数（最大4コマ）を計算してフラグに合う位置で停止
   ├ 引き込み制御:
   │    - state.phase === 'BONUS_STANDBY' のとき、内部成立中のBONUS図柄（赤7 or 赤赤青）を狙って目押しすれば揃う
   │    - 青7フラグ成立（BONUS中）: **目押し不要で自動的に青7を揃える**（§6.1.1）
   │    - 小役フラグ成立: 対応絵柄引込
   │    - BONUS_STANDBY中の「リーチ目」: 専用停止制御でリーチ目を出現させる
   └ 目押し失敗時: 小役は引込、レア役（チェリー等）はプレイヤーの目押し精度依存

5. 全停止後 → 払い戻し判定 (SlotEngine.evaluate)
   ├ payline上の絵柄並びを取得
   ├ payoutsテーブルと照合
   └ medalAdd(payout)

6. 状態遷移 (StateMachine.transition)
   ├ BONUS_STANDBY中・BONUS図柄揃い成立 → phase = 'BONUS'、BONUS消化開始
   ├ BONUS中・青7成立 → state.blue7Succeeded = true
   ├ BONUS中・BONUSゲーム数消化完了 → BONUS終了ロジック（§5.4 復帰処理）
   └ その他phase固有の遷移（NORMAL→ZENCHO、CZ→ART等）
```

### 6.1.1 青7チャレンジの挙動詳細（目押し不要の疑似遊技）

- BONUS中は毎G 青7フラグ抽選が走る（独立抽選）
- **青7フラグ成立ゲーム**: 
  - **プレイヤーの目押し操作を無視し、強制的に青青青で停止する疑似遊技として実装**
  - リール停止制御側で、全リールとも「青7が中段停止するコマ位置」に強制停止
  - プレイヤーは演出を見守るだけ（効果音・フラッシュ・祝福演出）
  - この仕様により、目押しスキルがないプレイヤーでも機械割と期待度が設計通りになる
- **未成立ゲーム**: 通常の小役抽選に従って停止
- BONUS中の1ゲームでも青7が揃えば、`state.blue7Succeeded = true` となりBONUS終了時にART付与

### 6.1.2 BONUS_STANDBY中の告知演出

- BONUS内部成立後、プレイヤーが図柄を揃えるまで続く状態
- **小役確率テーブルを専用の `standby` テーブルに差し替え**:
  - レア役（チェリー、スイカ、チャンス目）の確率を3〜5倍に増強
  - 「リーチ目」フラグを追加（通常時には存在しない、確定告知用の特殊出目）
- **リーチ目の表示**: 停止制御側で、例えば「左リール中段赤7→中リール中段赤7→右リール1コマ滑りで赤7停止」のような専用パターンを出現させる
- AUTO時: 図柄を自動的に揃える（BONUS_STANDBYに入ったら数ゲーム以内で確実にBONUS開始）

### 6.2 AUTO プレイ仕様

`CASINO_VISIBLE=true` 状態で AUTO トグルON の場合:
- BET → レバー → STOP1/2/3 すべて自動
- 押し順ナビがある場合は自動で正解順を選択
- BET 間隔は 0.5s（設定でさらに高速化オプション）
- メダル残高が3枚未満になったら自動停止

### 6.3 目押し仕様（手動プレイ）
- **通常時**: 小役（ベル、リプレイ）は100%引き込み。チェリー/スイカ/チャンス目は目押しが必要
- **ART中**: 押し順ナビ表示。ナビを外すと当該ゲームで**取りこぼしのみ発生**（G数没収なし）
- **ビタ押し**: BIG中の特定箇所で±0コマ止めに成功すると上乗せ

### 7.2.1 1ボーナスあたりのART期待度の算出

```

  + CZ経由の期待度（独立）
```

例（設定4、BIG中 青7確率1/56、BIG=30G）:
- 青7未成立確率 = (55/56)^30 ≈ 0.587
- 青7成立期待度 = 41.3%

これを全設定で 25〜50% 幅に収まるよう調整。

## 14. アセット方針

参考画像は **加工せずそのまま使用する**。サイズや統一感はCSS側で調整。

### 14.1 配置済みアセット

| ファイル | 絵柄 | 備考 |
|----------|------|------|
| `public/art/casino/symbols/bell.png` | ベル | そのまま使用 |
| `public/art/casino/symbols/watermelon.png` | スイカ | 「JAC」テキストは残っているがゲーム性に影響なし |
| `public/art/casino/symbols/cherry.png` | チェリー | そのまま使用 |
| `public/art/casino/symbols/replay.png` | リプレイ | そのまま使用 |
| `public/art/casino/symbols/big7_red.png` | 赤7 | 「BIG BONUS」テキストはそのまま（BIG揃い演出で整合） |
| `public/art/casino/symbols/blue7.png` | 青7 | 「BIG BONUS」テキストはそのまま（ゲーム性への影響なし、ART告知演出でカバー） |

### 14.2 ランタイムの参照

- シンボル画像は `data/symbols.js` の `SYMBOLS[id].image` から参照（`assetPath()` 経由で解決）
- `SlotRenderer.js` がこの SYMBOLS マップから画像URLを取得し、リール描画する
- 表示サイズは CSS の `width` / `height` と `object-fit: contain` で統一（例: 90px四方）

### 14.3 将来の拡張（任意）

- 余裕ができた段階で、統一感のあるアセットに差し替え検討
- アニメーション絵柄（例: BIG7の光沢）はスプライトシート化して `art/casino/symbols/animated/` に配置
- 高解像度版（Retina対応）は `@2x.png` サフィックスで対応
- プレミア演出用の特殊絵柄（虹7など）は後追いで追加
