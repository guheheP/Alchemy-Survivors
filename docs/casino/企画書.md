**ステータス**: 実験的機能（ユーザーには当面非表示）

## 1. コンセプト

Alchemy-Survivors のプレイヤー体験に、本編攻略とは**完全に別軸の娯楽**として「日本式パチスロ（A+ARTタイプ）」を提供する。錬金術師たちが裏世界で運営する闇の賭博場 **「錬金賭博場 (Alchemist's Den)」** として世界観に組み込む。

### 1.1 プレイヤー体験のコア
- 短時間（1分〜数十分）で完結する別ゲーム体験
- 目押し技術が上達すれば少し有利になる「スキル介入の余地」
- 本編経済を破壊しない **機械割 約100%**（長期収支 ±0）
- 「今日の戦利品、今日のうちに溶かそうか？」というフレーバー

### 1.2 スタンス
- **実験的機能**として明示。タイトルバナーに注釈を入れ、本編メタ進行の主経路ではないことを明言
- 本編ゴールド稼ぎより長期ROIが悪いよう調整し、「本編を回避する攻略法」にはならない設計
- 運営方針次第でいつでも削除可能（[設計書 §10 削除手順書](設計書.md#10-削除手順書) 参照）

## 2. 外部インターフェース（既存システムとの接点）

**ゴールド以外は一切依存しない**。以下の1点のみが既存システムとの接点。

| 接点 | 方向 | 用途 |
|------|------|------|
| `InventorySystem.addGold` / `spendGold` | casino → 本体 | 両替時のみ呼び出し |

カジノ遊技中は完全にメダル通貨で完結し、ゴールドには一切触らない。

## 4. 両替レート設計

| 項目 | 決定値 |
|------|--------|
| ゴールド → メダル | **1G = 1メダル** |
| メダル → ゴールド | **1メダル = 1G** |
| 両替手数料 | **0%**（ストレート往復） |
| 最小掛け枚数 | **3枚固定**（1ゲーム=3メダル=3G相当） |
| 端数処理 | 整数単位のみ受付、端数入力不可 |
| 両替下限 | 1回あたり 10G 以上から受付（誤操作防止） |

### 4.1 メダル表記

- 所持メダルは常にスロット画面上部に表示

## 5. 機種仕様: 「賢者の石を求めて」(In Search of Lapis)

A+ARTタイプのパチスロ。錬金術世界観に寄せた演出で、ART突入＝「賢者の石の精製開始」、BIG＝「火竜の鱗の錬成」、REG＝「水竜の鱗の錬成」。

### 5.1 絵柄対応表

| 絵柄ID | 絵柄 | 参考画像 | 用途 |
|--------|------|----------|------|
| `BELL` | ベル（黄銅片） | ![bell](../casino/参考アセット/ベル.png) | 最高頻度小役、3枚払い出し |
| `WATERMELON` | スイカ（月光草） | ![watermelon](../casino/参考アセット/スイカ.png) | レア小役、高配当 |
| `CHERRY` | チェリー（血色結晶） | ![cherry](../casino/参考アセット/チェリー.png) | 角/中段チェリー、レア役トリガー |
| `REPLAY` | リプレイ（賢者の石の欠片） | ![replay](../casino/参考アセット/リプレイ.png) | 再遊技 |
| `BIG7` | 赤7（火竜の鱗） | ![big7](../casino/参考アセット/赤7.png) | BIG揃い・REG異色揃い用 |
| `BLUE7` | 青7（水竜の鱗） | ![blue7](../casino/参考アセット/青7.png) | REG異色揃い・ART契機の青青青 |

### 5.1.1 役と絵柄の組み合わせ

| 役 | リール並び | 備考 |
|----|-----------|------|
| ベル揃い | `BELL BELL BELL` | 最高頻度、3枚払い出し |
| スイカ揃い | `WATERMELON WATERMELON WATERMELON` | レア小役 |
| 角チェリー | 左リールに `CHERRY` 停止（中右は任意） | レア役トリガー |
| リプレイ | `REPLAY REPLAY REPLAY` | 再遊技 |
| **チャンス目** | **特定の出目パターン**（§5.1.2） | 専用絵柄は用意せず、既存絵柄の停止形で表現 |
| BIG揃い | `BIG7 BIG7 BIG7`（赤赤赤） | BIGボーナス |
| REG揃い | `BIG7 BIG7 BLUE7`（赤赤青） | REGボーナス |
| 青7揃い | `BLUE7 BLUE7 BLUE7`（青青青） | ART契機（BONUS中のみ出現） |

### 5.1.2 チャンス目の扱い

チャンス目は **専用絵柄を用意せず、既存絵柄の特定停止パターン** として実装する。CHANCE フラグ成立時、停止制御側で以下のような「通常は出ない」特徴的な並びを生成する。

**候補パターン（実装時に確定）**:
- パターンA: 中段 `BELL - WATERMELON - CHERRY`（中段一直線の異種混合）
- パターンB: 右下がり `BIG7 - BLUE7 - BELL`（リーチ目に似た"外し"形）
- パターンC: 上段 `CHERRY - BELL - WATERMELON`

採用方針: プレイヤーが「あ、チャンス目だ」と一目で認識できる、通常フラグでは出現しない並びを1〜2種用意。リーチ目（BONUS_STANDBY中専用）とは異なるパターンにする。

### 5.1.3 アセット方針

参考画像は加工せずそのまま使用する。

- 配置先: `public/art/casino/symbols/`（配置済み）
- ファイル名: `bell.png` / `watermelon.png` / `cherry.png` / `replay.png` / `big7_red.png` / `blue7.png`
- サイズ: 各画像の元サイズを維持。CSS側で統一表示サイズに合わせる
- `blue7.png` の「BIG BONUS」テキスト: そのまま残す（ゲーム性への影響なし、ART契機の告知演出でカバー）

### 5.2 ボーナスとARTの関係（重要）

この機種は **ボーナス抽選とART抽選が独立** し、ARTの獲得経路が限定的なのが設計上の肝。

#### 5.2.1 ボーナス抽選は常時（内部成立→待機→揃えで確定）
- **BIG / REG は BONUS中・BONUS待機中以外のすべての状態で常時抽選**
  - NORMAL / ZENCHO / CZ / ART / TENJOU すべてで抽選が走る
  - 確率自体は状態を問わず一定（設定によってのみ変動）
- **BONUS内部成立時は即 BONUS へ遷移せず、BONUS待機状態（BONUS_STANDBY）へ移行**
  - 待機状態ではボーナス図柄が揃うまで通常ゲームを継続
  - 待機状態中は **CZ抽選・前兆抽選・天井カウント等の他の抽選は全停止**
  - 小役・リプレイは引き続き抽選・払い出しあり
- **内部成立の告知**: 待機状態では **レア役（チェリー／スイカ／チャンス目）やリーチ目が出現しやすくなる** 演出でプレイヤーに当選を示唆
- プレイヤーがボーナス図柄を揃えた時点で BONUS 状態へ遷移し、ボーナス開始

#### 5.2.2 ART突入経路（2つのみ）
ARTの突入経路は **以下の2経路のみ**。ボーナス単体ではARTに突入しない。

| 経路 | 契機 | 備考 |
|------|------|------|
| **CZ経由** | CZ成功時 | CZ中に成立する特定フラグ or 所定ゲーム数内のフラグ出現で成功 |
| **ボーナス中の青7揃い** | BONUS中の疑似遊技で青7が揃う | 目押し不要。内部フラグ成立時は自動で青7が揃う演出が発動 |

**トータルART期待度（1ボーナスあたり）**: **25〜50%**（設定により変動）

#### 5.2.3 ART中ボーナス = 大量上乗せチャンス
- ART中にBONUSが当選すると、BONUS待機→BONUS消化の間は **ART残G数の消化を凍結**
- BONUS終了後、ART復帰時に以下の上乗せが発生

| 結果 | 上乗せ |
|------|--------|
| 青7チャレンジ成功（ART中BONUS中に青7フラグ成立） | **+100G固定** ＋ ART再セット（ストック追加） |
| 青7チャレンジ失敗（BONUS中に青7フラグが成立しなかった） | 上乗せなし（ART残G数のみ復帰） |

- 「ART中にボーナスを引いて青7を狙う瞬間」を射倖性の核に据える

### 5.3 収支バランス方針

機械割の目標は設定4で **約100%**。設定1〜6 の分布:

| 設定 | 機械割 | 毎日の出現頻度 |
|------|--------|----------------|
| 1 | 96.5% | 30% |
| 2 | 98.0% | 25% |
| 3 | 99.5% | 20% |
| 4 | 100.5% | 15% |
| 5 | 102.0% | 5% |
| 6 | 106.0% | 5% |

**設定システム**: 設定は**毎日ランダム**で決定（ゲーム内時刻ベース）。プレイヤーへの告知はなし（察しプレイ）。同じ日に複数セッションを回しても設定は不変。

**設定4 の配分内訳（目安）**:

| 収支項目 | 比率 | 備考 |
|----------|------|------|
| 通常時の吸い込み | -42% | 通常ゲーム中のコイン持ち |
| 小役払い戻し（ベル等） | +22% | 通常時の出玉ベース |
| BIG出玉（常時抽選） | +20% | 約1/300、1回 約250枚 |
| REG出玉（常時抽選） | +8% | 約1/450、1回 約80枚 |
| ART出玉（CZ経由＋青7経由） | +10% | 突入率25-50%／継続G数／上乗せで調整 |
| ART中ボーナス上乗せ | +2.5% | ART中BONUSによる継続・ストック |
| **合計** | **約100.5%** | |

**重要**: ART単独の出玉貢献は控えめ（+10〜13%）。「ART中にボーナスを引く」瞬間が最大ハイライトになるよう配分し、爆発力と堅実性を両立させる。

## 6. プレイサイクル詳細（プレイヤー目線）

### 6.1 典型的な出玉フロー

```
[入店] 両替所で 300G → 300メダル
  ↓
[通常時] 3枚掛けで回す。ベル多め。常時ボーナス抽選＋レア役でCZ抽選
  ├→ [BONUS内部成立] 当選後は BONUS待機 へ
  ↓
[前兆発生] チャンス目で内部当選疑似。錬成陣が薄く浮かぶ
  ↓
[CZ突入] 10G間のチャンスゾーン。成功率 約40%
  ↓ (成功) ────────────────────────────┐
  ↓ (失敗) → 通常時復帰                 ↓
                                    [ART突入]

[BONUS待機] レア役・リーチ目が頻発し、ボーナス当選を告知
  └→ プレイヤーがボーナス図柄を揃えると BONUS開始

[BONUS（疑似遊技）] 規定ゲーム数のボーナス消化。青7チャレンジ抽選が走る
  └→ 青7フラグ成立 → 自動で青7揃い演出 → ART確定

[ART] 50Gの初期ゲーム数、純増2枚/G
  ├ 押し順ナビに従ってメダル獲得
  ├ レア小役で +10〜100G 上乗せ抽選
  └ ★ART中のBONUS当選 → ART残G数凍結 → BONUS消化★
        ↓
     [BONUS中] 青7チャレンジ抽選
        ├→ 青7フラグ成立 → 自動で揃う → +100G固定＋ストック追加
        └→ 青7不成立    → 上乗せなし
        ↓
     [ART復帰] 上乗せ分だけG数が伸びている
  ↓
[ART終了] 合計 100〜数百Gで終了、大量メダル獲得
  ↓
[通常時復帰] … 以降繰り返し

[天井] 1200G ハマりで強制ボーナス内部成立（待機経由で放出）
```

### 6.2 ART突入経路のまとめ（プレイヤー視点）

1. **王道ルート**: 通常時 → レア役で前兆 → CZ → 成功 → ART
2. **一撃ルート**: 通常時／ART中にBONUS内部成立 → BONUS中に青7成立 → ART（またはART継続＋100G上乗せ）
3. **救済ルート**: 1200Gハマリで強制BONUS → 青7チャレンジ

1周期: **数分〜30分**。短いセッションで完結。

### 6.3 操作系

| 操作 | 方式 |
|------|------|
| BET | 3枚固定。ボタン1回で3枚投入 |
| レバー | クリック or スペースキーで回転開始 |
| STOP | STOP1 / STOP2 / STOP3 の3ボタン |
| AUTO | トグル。ONの間は **完全自動**（BET → 回転 → 停止まで） |

### 6.4 目押しとナビ

- **通常時**: ベル等の小役は目押し不要（100%引き込み）。チェリー等は目押し成功で取りこぼし回避
- **BONUS待機中**: ボーナス図柄を揃えるための目押しが必要（AUTO時は自動で揃える）
- **BONUS中の青7チャレンジ**: **目押し不要の疑似遊技**。内部フラグ成立時は自動で青7が揃う演出が発動。プレイヤーは演出を見守るだけ
- **ART中**: 押し順ナビを画面に表示。ナビを外すと**該当ゲームで取りこぼし**のみ発生（ペナルティG没収なし）

## 7. UI/画面遷移

```
[拠点タブ] 「🎰 賭博場」タブ（初期は非表示、CASINO_VISIBLE=true で出現）
  ↓
[CasinoScreen ロビー] 機種リスト、両替所、戻るボタン、所持メダル表示
  ├ [ExchangeDialog] モーダルで両替
  └ [SlotScreen] スロット画面へ遷移
       ├ リール3つ
       ├ BETボタン（3枚固定）
       ├ レバー（START ボタン）
       ├ STOP1 / STOP2 / STOP3 ボタン
       ├ AUTO トグル
       ├ 情報パネル（現在G数、天井までの残り、ART残り、履歴）
       ├ 演出レイヤー（前兆/CZ/ボーナス/ART）
       └ 戻る（ロビーへ）
```

### 7.1 タブ位置と可視性

- **配置場所**: 既存タブの特定位置に挿入（`shop` の隣を予定）
- **可視性フラグ**: `CASINO_VISIBLE`
  - **初期値 `false`**: タブ非表示。ユーザーからはアクセス不能
  - 開発者は `localStorage.setItem('casino_visible', '1')` や URLパラメータ `?casino=1` でデバッグ表示可能
  - 正式リリース時にフラグを `true` に切り替えるだけで全ユーザーに公開される
- `CASINO_ENABLED`（機能そのもののON/OFF）と `CASINO_VISIBLE`（UIの露出）は独立

### 7.2 初回チュートリアル

- 初めてカジノ画面を開いた時、段階的チュートリアル（3ステップ程度）を表示
- スキップ可能、再表示ボタンあり
- AUTO プレイを推奨してパチスロ未経験者への敷居を下げる

## 9. オミット戦略（削除時の扱い）

| 削除対象 | プレイヤーから見た扱い | 技術的処理 |
|----------|------------------------|------------|
| 所持メダル残高 | **全額ゴールドに自動還付** | 削除バージョン適用時にワンショットマイグレーションで `addGold(medals)` |
| ART中の残りG | 破棄 | 無視（エフェメラル状態のため） |
| 設定・履歴 | 破棄 | セーブから除去 |
| 累計獲得メダル/累計投資 | 破棄 | セーブキーごと削除 |
| ゴールド残高 | 影響なし（最重要） | `InventorySystem` は無変更 |

### 9.1 削除フロー
1. フラグ `CASINO_ENABLED = false` に切替 → タブ消失、state読み込みなし
2. 次回ロード時、セーブデータ内の `casino.medals` をゴールドに変換（ワンショット）
3. `src/game/casino/` ディレクトリごと物理削除可能（[設計書 §10](設計書.md#10-削除手順書)）

## 11. リスク管理

### 高リスク
- **機械割の実測乖離**: 確率テーブル設計ミスで±5%以上ずれる → シミュレータで10万G×全設定を自動検証、CIでガード
- **パチスロ未経験プレイヤーへの操作説明**: 目押し・ナビ・設定等、用語が多い → 段階的チュートリアル、ヘルプボタン常駐、AUTO推奨をデフォルト
- **依存の漏れ**: 疎結合が知らぬ間に破れる → `import` ルール厳守、PR時に `src/game/casino/` 外への書き込みを目視確認

### 中リスク
- セーブ肥大化 → `lifetimeStats` のサイズ上限を1KB以内で監視
- RNGのバイアス → `util/rng.js` で Mulberry32 等の良質RNG使用
- モバイル操作性 → 3STOP + AUTO + BET の密集ボタンに大きめタップ領域

### 低リスク
- CSS衝突 → `.casino-*` プレフィックス徹底
- イベント名衝突 → `casino:*` プレフィックス徹底
- 画像アセット → `public/assets/casino/` に閉じ込め
