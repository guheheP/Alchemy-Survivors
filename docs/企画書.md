## 2. コンセプト

**「自分で鍛えた武器で戦場を制圧する快感」**

Vampire Survivors系ゲームの課題は「武器獲得がランダムすぎて戦略を立てにくい」こと。本作はラン間の錬金術クラフトで装備を自作し、品質・特性（トレイト）を戦略的に厳選してからランに臨む。

- **ラン中**: 移動操作のみ。装備した武器が自動で攻撃。レベルアップ時にパッシブ選択。
- **ラン間**: 素材を使って錬金術でウェポン/アーマー/アクセサリを調合。テトリスパズルミニゲームで品質ボーナスを狙う。
- **特性システム**: 錬金で付与した特性がラン中のパッシブ能力に変換される（例: 「攻撃力+++」→常時ATK+28、「再生」→HP2/秒自動回復、「疾風」→移動速度+20%）。

**核心的差別化**: 他のヴァンサバ系ではラン中にランダムで武器を拾うが、本作では**事前に鍛えた装備を持ち込む**。クラフトの深さ（80+レシピ、34特性、融合チェーン）がそのままビルドの深さになる。

## 4. コアゲームプレイ

### 4.1 ラン中ゲームプレイ（サバイバルアクション）

#### 基本操作

- **移動**: WASD / 矢印キー / モバイルは仮想スティック
- **攻撃**: 完全自動（装備した武器種に応じた攻撃パターン）
- **ダッシュ**: Spaceキー（短距離回避、クールダウンあり）

#### 武器種別の自動攻撃パターン

| equipType | 攻撃パターン | 射程 | 特徴 |
|-----------|-------------|------|------|
| `sword` | 前方扇型斬撃 | 近距離 | 高ダメージ、狭範囲 |
| `spear` | 前方直線突き | 中距離 | 貫通、中ダメージ |
| `bow` | 最寄り敵に矢を発射 | 遠距離 | 単体、高速連射 |
| `staff` | 周囲にオーブ発射 | 中距離 | 全方位、魔法属性 |
| `dagger` | 高速連続斬り | 極近距離 | 超高速、低単発 |
| `shield` | 周囲にバリア波動 | 近距離 | ノックバック、低ダメージ |

#### 品質 → ラン内ステータス変換

既存の品質システム (0-999) をそのまま活用。品質が武器の基本性能を決定する。

```
武器の攻撃力 = baseValue / 10 + quality / 5
攻撃速度     = 1.0 + quality / 500  (最大約3.0倍)
攻撃範囲     = baseRange * (1 + quality / 1000)
```

防具(armor/robe/shield):
```
ダメージ軽減 = baseValue / 12 + quality / 8
最大HP増加   = quality * 0.5
```

アクセサリ:
```
移動速度増加 = baseValue / 15 + quality / 10
特殊効果倍率 = 1 + quality / 200
```

#### 特性 → パッシブ変換テーブル

既存34特性をラン中パッシブとしてマッピング。既存の `effects` フィールドを拡張し、ラン用効果を追加定義。

| 既存特性 | ラン中パッシブ効果 |
|---------|-------------------|
| `闘志` (battleAtk: 3) | 武器ダメージ+3 |

| 時間 | イベント |
|------|---------|

#### レベルアップシステム

敵を倒して経験値ジェムを集め、レベルアップ時に3択からパッシブを選択（Vampire Survivors方式）。

パッシブ候補例:
- **ダメージ+10%**: 全武器のダメージを増加
- **攻撃範囲+15%**: 全武器の範囲を拡大
- **HP+20**: 最大HPを増加
- **移動速度+8%**: 移動速度を増加
- **マグネット範囲+30%**: 経験値ジェムの吸引範囲を拡大
- **追加射出+1**: 武器の発射数を+1（bow, staffに特に有効）
- **クールダウン-10%**: 武器の攻撃間隔を短縮
- **リジェネ+1/秒**: HP自動回復を追加
- **クリティカル率+5%**: クリティカルヒット確率

#### 消耗品の活用

既存の `consumable` アイテムをラン中の限定使用アイテムとして持ち込み可能。

- 各ラン開始時にアクティブアイテムスロット2-3枠を選択
- 手動発動（数字キー 1-3）
- 使用回数制限あり（既存の `battleEffect.uses` + 特性 `battleItemUses` を流用）
- `battleEffect` データをそのまま活用: heal, buff, debuff, stun, damage, revive

### 4.2 クラフトゲームプレイ（既存システム流用）

#### 錬金工房（拠点）

既存のクラフトシステムをほぼそのまま使用。

1. **レシピ選択**: 80+レシピから作りたい装備を選ぶ
2. **素材配置**: カテゴリスロット (`@ore_type`, `@herb_type` 等) に手持ち素材をセット
3. **パズルミニゲーム**: 4x4グリッドにテトリス型ブロックを配置し、隣接ボーナスで品質を上乗せ
4. **特性選択**: 素材の特性を引き継ぎ、同一特性x2で融合（Common→Uncommon→Rare→Epic）
5. **完成品獲得**: 品質・特性が確定した装備品を取得

### 5.3 技術的な主要課題

#### A. Canvas 2D パフォーマンス（数百体同時描画）

- **空間ハッシュグリッド**: 衝突判定を O(n²) → O(n) に削減（セルサイズ64px）
- **オブジェクトプール**: 敵・弾丸・エフェクトを事前確保しGCを回避
- **ビューポートカリング**: 画面外エンティティの描画をスキップ
- **バッチ描画**: 同一スプライトをまとめて drawImage
- **固定タイムステップ**: 60fps物理 + 可変fps描画の分離

```javascript
// ゲームループの基本構造
const FIXED_DT = 1 / 60;
let accumulator = 0;

function gameLoop(timestamp) {
  const dt = (timestamp - lastTime) / 1000;
  lastTime = timestamp;
  accumulator += dt;

  while (accumulator >= FIXED_DT) {
    update(FIXED_DT);    // 物理・衝突・AI（固定60fps）
    accumulator -= FIXED_DT;
  }

  render(accumulator / FIXED_DT);  // 補間付き描画（可変fps）
  requestAnimationFrame(gameLoop);
}
```

#### B. 既存TraitDefsの拡張

既存の `effects` に `run` プレフィックス付きフィールドを追加。既存機能には影響なし。

```javascript
// 拡張例
'攻撃力+++': {
  name: '攻撃力+++', rarity: 'epic',
  effects: {
    battleAtk: 28,        // 既存（維持）
    runDamageFlat: 28,    // ★新規: ラン中武器ダメージ+28
  }
},
'疾風': {
  name: '疾風', rarity: 'epic',
  effects: {
    speedBonus: 20, exploreSuccess: 8,  // 既存（維持）
    runMoveSpeed: 0.20,                  // ★新規: 移動速度+20%
    runDodge: 0.08,                      // ★新規: 回避率+8%
  }
},
```

#### C. ボスのリアルタイムアクション変換

| 既存フィールド | ラン中の挙動 |
|---------------|-------------|
| `maxHp`, `atk`, `def`, `spd` | そのまま使用（スケーリング係数あり） |
| `phases[].hpThreshold` | HP%到達時にモード変化 |
| `phases[].effect` | ステータスバフ発動 |
| `skills[].type: 'attack'` | 突進攻撃（単体） |
| `skills[].type: 'aoe'` | 円形範囲攻撃 |
| `skills[].type: 'heavy'` | 叩きつけ（高ダメ、予告表示あり） |
| `skills[].type: 'heal'` | 一定時間停止してHP回復 |
| `skills[].chance` | 各攻撃の選択確率（既存のまま） |

## 8. リスクと対策

| リスク | 影響度 | 対策 |
|--------|-------|------|
| Canvas 2Dで数百体の敵描画がフレーム落ち | 高 | 空間ハッシュ+オブジェクトプール+カリング。最悪WebGLに段階移行 |
| クラフトUIがアクションゲームと馴染まない | 中 | 拠点画面をダーク系UIにリデザイン |
| ヴァンサバとクラフトの体験が分断される | 高 | 特性→パッシブ変換を充実させ、「この特性のために素材を集めたい」動線を強化 |
| モバイルブラウザのパフォーマンス不足 | 中 | 敵上限数のデバイス別調整（PC:500体、モバイル:200体） |
| 品質0-999の幅が広すぎてバランス困難 | 中 | 品質→ステータス変換に対数カーブを導入 |

## 9. 差別化ポイント

| 要素 | 一般的なVS系 | 本作 |
|------|-------------|------|
| **武器獲得** | ラン中にランダムで出現 | ラン前にクラフトで自作 |
| **ビルド戦略** | ラン中の運次第 | 事前に特性・品質を計算して装備をビルド |
| **素材の意味** | なし（経験値のみ） | クラフト素材として永続的価値がある |
| **パズル要素** | なし | テトリス型パズルで品質ボーナス |
| **キャラ差** | 初期武器が違う程度 | ジョブごとの装備制限とパッシブの組み合わせ |
| **メタ進行** | コイン→恒久アップグレード | 素材蓄積→クラフト→装備強化→新エリア |
| **リプレイ動機** | 「いい武器が出るかも」 | 「あの素材でEpic特性を融合したい」 |

### 本作のユニークバリュー

1. **「準備の楽しさ」と「戦闘の爽快感」の融合** — アトリエの錬金術 + Vampire Survivorsの手軽さ
2. **特性融合チェーンによるビルドの奥深さ** — Common→Epic 4段階融合の戦略性
3. **テトリスパズルというスキル要素** — クラフト品質がプレイヤースキルで変わる
4. **80+レシピのクラフト深度** — 実証済みの膨大なコンテンツ量でスタート
